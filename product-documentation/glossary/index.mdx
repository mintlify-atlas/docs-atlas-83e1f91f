Here are some terms used throughout incident.io's products and documentation, most should be self-explanatory. Refer to the relevant section for further details.

## A

<AccordionGroup>
  <Accordion title="@incident">
    Chat bot in Slack channels to manage incidents, draft updates, and answer questions
  </Accordion>
  <Accordion title="Action">
    Something that needs to be done during the active phase of an incident to mitigate or communicate (cf. Follow-up)
  </Accordion>
  <Accordion title="Alert">
    Automated event from a monitoring system signaling something unusual or problematic was detected
  </Accordion>
  <Accordion title="Alert attributes">
    Meta-tagging system on alerts, similar to custom fields; can use catalog
  </Accordion>
  <Accordion title="Alert route">
    Configuration that determines what to do with incoming alerts
  </Accordion>
  <Accordion title="Alert source">
    Integration that sends alerts (monitoring systems, webhooks, email, etc.)
  </Accordion>
  <Accordion title="Alias">
    Alternate name for an entry referenced in other systems
  </Accordion>
  <Accordion title="Announcement">
    Dynamic post announcing that an incident has happened
  </Accordion>
</AccordionGroup>

## C

<AccordionGroup>
  <Accordion title="Call routing">
    Phone number setup to directly call users or create incidents
  </Accordion>
  <Accordion title="Catalog">
    Connected map of everything in your organization; integrates context and structure to power automations and reporting
  </Accordion>
  <Accordion title="Catalog Attribute">
    Information that correlates to an entry (like a column); can be text, number, boolean, or another catalog type
  </Accordion>
  <Accordion title="Catalog Entry">
    A single item in a catalog type (like a row)
  </Accordion>
  <Accordion title="Catalog Type">
    A list of items in the catalog (like a spreadsheet); e.g., Services, Teams, Products
  </Accordion>
  <Accordion title="Components">
    Parts of your system displayed on status pages or used in incident attributes
  </Accordion>
  <Accordion title="Condition">
    Function that determines when an automation runs (if X then Y)
  </Accordion>
  <Accordion title="Contact methods">
    How someone wants to be contacted when paged (e.g., phone, SMS, push)
  </Accordion>
  <Accordion title="Cover request">
    Ask for someone on the schedule to take all or part of your shifts
  </Accordion>
  <Accordion title="Custom field">
    Additional metadata field on incidents (can be single/multi-select, number, URL, or text)
  </Accordion>
</AccordionGroup>

## D

<AccordionGroup>
  <Accordion title="Debrief">
    Meeting held after an incident is resolved to discuss learnings
  </Accordion>
  <Accordion title="Decision flow">
    Decision tree that can be triggered by a workflow
  </Accordion>
  <Accordion title="Deduplication key">
    Unique identifier that prevents duplicate alerts from firing multiple times
  </Accordion>
  <Accordion title="Derived attribute">
    Automatically filled attribute based on relationships to other types
  </Accordion>
  <Accordion title="Duration metric">
    Time between two timestamps (e.g., time to resolution)
  </Accordion>
</AccordionGroup>

## E

<AccordionGroup>
  <Accordion title="Escalation">
    Mechanism to page someone; automated or manual request to get a hold of one or more people
  </Accordion>
  <Accordion title="Escalation path">
    Series of steps defining how to page people; connected to teams or catalog objects
  </Accordion>
  <Accordion title="Expression">
    Conditions and catalog-based logic that power workflows and dynamic fields
  </Accordion>
</AccordionGroup>

## F

<AccordionGroup>
  <Accordion title="Follow-up">
    Post-incident action item to improve detection, remediation, or prevention for future incidents (cf. Action)
  </Accordion>
  <Accordion title="Form">
    Interface for declaring, updating, resolving, or escalating incidents
  </Accordion>
</AccordionGroup>

## G-H

<AccordionGroup>
  <Accordion title="Grouping">
    Sending related subsequent alerts into an incident based on time and/or attributes
  </Accordion>
  <Accordion title="Handover">
    Passing of a role from one person to another
  </Accordion>
</AccordionGroup>

## I

<AccordionGroup>
  <Accordion title="Incident">
    A place for humans and machines to coordinate a response when things break
  </Accordion>
  <Accordion title="Incident Lifecycle">
    Set of statuses and timestamps that define incident progression
  </Accordion>
  <Accordion title="Incident Mode">
    Classification of an incident: Standard, Test, Tutorial, or Retrospective
  </Accordion>
  <Accordion title="Incident Status">
    Current state of an incident within the lifecycle (e.g., Triage, Active, Resolved, Closed)
  </Accordion>
  <Accordion title="Incident Summary">
    Detailed description of what the incident is about
  </Accordion>
  <Accordion title="Incident ticket">
    Auto-created Jira/Linear/ServiceNow (and many others) ticket associated with an incident that stays in sync
  </Accordion>
  <Accordion title="Incident Timeline">
    Events that happened during the incident, including interactions with the app and pinned posts
  </Accordion>
  <Accordion title="Incident Timestamp">
    Date/time marking moments in the lifecycle (core or custom)
  </Accordion>
  <Accordion title="Incident Type">
    Wrapper of configuration that segments incident use cases and processes
  </Accordion>
  <Accordion title="Incident Update">
    Communication on what is happening now in an incident
  </Accordion>
  <Accordion title="Incident Visibility">
    Whether an incident is Public or Private
  </Accordion>
  <Accordion title="Insights">
    Reporting and analytics dashboards
  </Accordion>
</AccordionGroup>

## L-O

<AccordionGroup>
  <Accordion title="Layer">
    Concurrent users scheduled at the same time in a single rota
  </Accordion>
  <Accordion title="Observers">
    Incident participants not active in the Slack channel
  </Accordion>
  <Accordion title="Override">
    Substitute a person on a shift or part of a shift
  </Accordion>
</AccordionGroup>

## P

<AccordionGroup>
  <Accordion title="Participants">
    People involved in an incident; includes roles and active/observer participants
  </Accordion>
  <Accordion title="Policy task">
    Task assigned when someone is out of compliance with a policy
  </Accordion>
  <Accordion title="Post-incident task">
    Task to complete after Resolution before Closing (e.g., review timeline, create post-mortem)
  </Accordion>
  <Accordion title="Post-mortem">
    Document for learnings after an incident (also called Post-Incident Review, RCA)
  </Accordion>
  <Accordion title="Priority">
    How important an alert or escalation is; determines if it should be handled during working hours or urgently out of hours
  </Accordion>
</AccordionGroup>

## Q-R

<AccordionGroup>
  <Accordion title="Query">
    Dynamic values powered by catalog; route through attributes to return values
  </Accordion>
  <Accordion title="Role">
    Directly responsible person during an incident (e.g., Lead)
  </Accordion>
  <Accordion title="Rota">
    A rotation of one person or group; a schedule can have many rotas
  </Accordion>
</AccordionGroup>

## S

<AccordionGroup>
  <Accordion title="Schedule">
    A person or group made available at certain times to be paged or added to a Slack user group
  </Accordion>
  <Accordion title="SCIM">
    Use Okta to control provisioning of on-call seats or user roles
  </Accordion>
  <Accordion title="Scribe">
    AI note taker that joins incident calls, transcribes, and provides summaries and key moments
  </Accordion>
  <Accordion title="Seat">
    License type: Responder or On-call. Refer to Billing.
  </Accordion>
  <Accordion title="Severity">
    How bad an incident is; impact and urgency level (e.g., Critical, Major, Minor)
  </Accordion>
  <Accordion title="Shift">
    Single duration that a person is on-call before it changes to the next person
  </Accordion>
  <Accordion title="Shoutout">
    Kudos or gratitude sent to someone in a Slack channel
  </Accordion>
  <Accordion title="Slack user group">
    Group synced with a schedule; adds whoever is currently on-call
  </Accordion>
  <Accordion title="Stream">
    Child channel that creates a workstream for large incidents or to segment working groups
  </Accordion>
  <Accordion title="Suggestions">
    Context-based AI recommendations in channels and calls (e.g., summaries, follow-ups)
  </Accordion>
</AccordionGroup>

## T-U

<AccordionGroup>
  <Accordion title="Template">
    Reusable format for incident tickets, follow-ups, or post-mortems
  </Accordion>
  <Accordion title="User role">
    Set of permissions in the app (not to be confused with incident roles)
  </Accordion>
</AccordionGroup>

## V-W

<AccordionGroup>
  <Accordion title="Variable">
    Objects from incidents, alerts, follow-ups, users, and escalations used to dynamically insert text or set conditions
  </Accordion>
  <Accordion title="Webhook">
    Fires information when specific events happen
  </Accordion>
  <Accordion title="Workflow">
    Automation created to run communications, set roles, escalate, and more
  </Accordion>
  <Accordion title="Working hours">
    Days and times for shifts when people should be on-call
  </Accordion>
  <Accordion title="Workload">
    Time spent on incidents by responders, calculated from Slack activity and call time
  </Accordion>
</AccordionGroup>
