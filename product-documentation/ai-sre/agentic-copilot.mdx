# Agentic co-pilot and natural language interaction

The AI co-pilot enables natural language interaction with incident.io, allowing you to query incident data, retrieve context, update incidents, and perform common actions through conversational commands. Instead of navigating UIs or remembering exact commands, simply ask the co-pilot what you need in plain English.

<Info>
  The AI co-pilot is currently in early access. Features and capabilities are actively expanding. Contact [help@incident.io](mailto:help@incident.io) to request access.
</Info>

## What is the AI co-pilot?

The AI co-pilot is an agentic assistant that understands incident response context and can take actions on your behalf. Unlike simple chatbots that only answer questions, the co-pilot can:

- **Query incident history** - "Show me all database incidents from last quarter"
- **Retrieve specific information** - "What was the resolution time for INC-1234?"
- **Update incidents** - "Update this incident to SEV-2 and page the database team"
- **Execute workflows** - "Create a post-mortem for yesterday's outage"
- **Provide recommendations** - "What should we try next for this incident?"
- **Explain complex data** - "Why is our MTTD increasing?"

The co-pilot works in both Slack and the incident.io dashboard, meeting you where you already work.

### Understanding intent

- "What should we try" â†’ Suggest mitigation strategies

- "What's the best way" â†’ Provide guidance based on history

### Multi-step reasoning

The co-pilot breaks complex requests into steps:

**Request:** "Create a summary of all SEV-1 incidents from last month and send it to #engineering"

**Co-pilot reasoning:**
1. Query incidents with SEV-1 severity from last month
2. Extract key information (dates, duration, root causes)
3. Generate summary text
4. Format for Slack
5. Post to #engineering channel

**Result:** Summary posted with link to full incident list

### Context awareness

The co-pilot maintains context throughout conversations:

```
You: Show me recent database incidents
Co-pilot: [Displays 5 recent database incidents]

You: Which one took the longest to resolve?
Co-pilot: INC-1842 took 3 hours 45 minutes

You: What was the root cause?
Co-pilot: Connection pool exhaustion caused by inefficient queries

You: Show me the post-mortem
Co-pilot: [Displays post-mortem document link]
```

The co-pilot remembers previous queries in the conversation, so you can ask follow-up questions naturally.

## Using the co-pilot in Slack

Interact with the co-pilot directly in Slack:

### Invoking the co-pilot

**In incident channels:**
- Tag the bot: `@incident.io what should we try next?`
- The co-pilot responds with context-aware suggestions based on the current incident

**In DMs:**
- Send a direct message to the incident.io bot
- Ask any question about incidents, metrics, or perform actions
- No special syntax required

**In any channel:**
- Use `/inc ask` command: `/inc ask what were our SEV-1 incidents last week?`
- Results appear as ephemeral messages (only you see them)

## Co-pilot capabilities and limitations

### What the co-pilot does well

- **Natural language understanding** - Interprets varied phrasings of the same request
- **Context retention** - Remembers conversation history for follow-up questions
- **Data aggregation** - Combines information from multiple sources
- **Pattern recognition** - Identifies trends and similarities across incidents
- **Action execution** - Performs tasks that would require multiple UI clicks

### Current limitations

- **Complex multi-step workflows** - Some advanced workflows still require manual steps
- **Real-time system access** - Can't directly query live production systems
- **Ambiguity handling** - May ask clarifying questions if requests are unclear
- **External tool integration** - Limited integration with non-incident.io tools

<Info>
  Co-pilot capabilities are expanding rapidly during early access. Features that are limited today may be fully supported soon.
</Info>

## Configuring the co-pilot

Customize co-pilot behavior for your organization:

### Access control

Define who can use the co-pilot:

1. Navigate to **Settings â†’ AI â†’ Co-pilot**
2. Choose access level:
   - **All users** - Anyone in your organization
   - **Responders and admins** - Users with active incident permissions
   - **Admins only** - Limited to organization administrators

### Response preferences

Configure how the co-pilot responds:

- **Verbosity** - Brief vs. detailed responses
- **Proactiveness** - How often co-pilot offers unsolicited suggestions
- **Confirmation requirements** - Whether actions require explicit confirmation

### Data access

Control what data the co-pilot can access:

- **Historical incidents** - How far back to search
- **Sensitive incidents** - Whether to include private or restricted incidents
- **External data** - Which integrations co-pilot can query

### Custom commands

Create organization-specific shortcuts:

```
Command: "weekly incident summary"
Action: Generate summary of past week's incidents and post to #engineering

Command: "escalate to leadership"
Action: Update severity to SEV-1 and notify executive on-call
```

Custom commands enable team-specific workflows through simple phrases.

## Privacy and security

### What data the co-pilot accesses

- **Incident data** - All incident information you have permission to view
- **Integration data** - Data from connected tools (based on your permissions)
- **User data** - Your organization's user list and roles
- **Conversation history** - Previous co-pilot interactions (stored temporarily)

### Data handling

- Queries are processed using AI models (typically OpenAI or Anthropic)
- Your data is not used for model training
- Conversation history is retained for context but expires after 24 hours
- All standard incident.io access controls apply to co-pilot queries
- Co-pilot can only access data you have permission to view

### Disabling the co-pilot

Organizations can disable the co-pilot at any time:

1. Navigate to **Settings â†’ AI â†’ Co-pilot**
2. Toggle off **Enable co-pilot**
3. Co-pilot becomes unavailable; existing conversation history is deleted

For complete privacy information, see [AI Privacy Guidance](https://trust.incident.io/).

## Best practices

<Tip>
  **Start with simple queries** - Begin with straightforward questions to learn how the co-pilot interprets your requests, then progress to complex multi-step commands.
</Tip>

<Tip>
  **Be specific** - Include relevant details (time ranges, service names, severities) to get precise results. "Show me incidents" is broad; "Show me SEV-2 database incidents from last week" is specific.
</Tip>

<Tip>
  **Use follow-ups** - Ask follow-up questions to drill deeper. The co-pilot maintains context, so you don't need to repeat information.
</Tip>

<Tip>
  **Verify before executing** - When the co-pilot suggests actions, review before confirming. AI is powerful but not infallible.
</Tip>

<Tip>
  **Provide feedback** - Use thumbs up/down on co-pilot responses to help improve accuracy and relevance over time.
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Co-pilot doesn't understand my question">
    If the co-pilot struggles to understand:
    - Rephrase using more specific language
    - Break complex requests into smaller steps
    - Use incident.io terminology ("SEV-1" rather than "P0", "incident" rather than "outage")
    - Provide more context (time ranges, service names, etc.)
    
    You can also say "I don't think you understood" and rephrase.
  </Accordion>

  <Accordion title="Co-pilot response is incorrect">
    AI can make mistakes. When responses are wrong:
    - Use thumbs down to flag the error
    - Provide feedback explaining what was incorrect
    - Try rephrasing your question
    - For critical information, verify with the UI
    
    Flagged errors help improve the co-pilot for everyone.
  </Accordion>

  <Accordion title="Co-pilot is too slow">
    Response time depends on query complexity. If consistently slow:
    - Break complex queries into smaller requests
    - Use more specific queries (reduces data to process)
    - Check if incident.io API is experiencing latency
    
    Typical response time is 2-5 seconds. Longer than 10 seconds may indicate an issue.
  </Accordion>

  <Accordion title="Co-pilot can't perform an action I requested">
    If the co-pilot says it can't do something:
    - Confirm the action is supported (see capabilities section)
    - Check that you have the required permissions
    - Try breaking the action into smaller steps
    - Some actions may need to be done manually in the UI
    
    Co-pilot capabilities are expanding; actions not supported today may be added soon.
  </Accordion>

  <Accordion title="How do I clear co-pilot conversation history?">
    Conversation history automatically expires after 24 hours. To manually clear:
    - In Slack: Start a new DM thread
    - In dashboard: Close and reopen the co-pilot sidebar
    - History deletion doesn't affect incident data, only temporary conversation context
  </Accordion>
</AccordionGroup>
