## Key account features

<AccordionGroup>
  <Accordion title="Audit Logs" icon="clock-rotate-left">
    Track all actions and changes within your incident.io account for compliance and security.
    
    **What you can audit:**
    - User access and authentication
    - Configuration changes
    - Incident actions and updates
    - Integration activities
    - Permission modifications
    
    [Learn about audit logs →](/help-centre/account-admin/audit-logs)
  </Accordion>

  <Accordion title="Active Participants vs Observers" icon="users">
    Understand the difference between active participants and observers for billing and access purposes.
    
    **Key differences:**
    - **Active Participants**: Full access to create and manage incidents
    - **Observers**: Read-only access to view incidents and reports
    - Billing implications
    - Permission levels
    
    [Learn about user types →](/help-centre/account-admin/active-participants-vs-observers)
  </Accordion>

  <Accordion title="Permissions Management" icon="user-lock">
    Configure granular permissions to control what users can do within incident.io.
    
    **Permission areas:**
    - Incident management
    - Configuration and settings
    - Integrations
    - Reporting and insights
    - Administrative functions
    
    [Explore permissions →](/help-centre/account-admin/permissions/index)
  </Accordion>

  <Accordion title="SAML Single Sign-On" icon="shield-halved">
    Configure SAML-based single sign-on for enhanced security and centralized authentication.
    
    **Benefits:**
    - Centralized authentication
    - Enhanced security
    - Simplified user management
    - Compliance requirements
    
    [Set up SAML →](/help-centre/account-admin/security-data-privacy/sign-in-with-saml)
  </Accordion>
</AccordionGroup>

## User management

<Tabs>
  <Tab title="User Types">
    **Active Participants**
    - Full incident management access
    - Can declare and manage incidents
    - Count toward billing
    
    **Observers**
    - Read-only access
    - View incidents and reports
    - Lower cost or free (plan dependent)
    
    [Learn more →](/help-centre/account-admin/active-participants-vs-observers)
  </Tab>
  <Tab title="Duplicate Users">
    If you encounter duplicate user errors, it usually means the same person has multiple accounts.
    
    **Resolution steps:**
    - Identify duplicate accounts
    - Merge or remove duplicates
    - Update integrations
    
    [Fix duplicate users →](/help-centre/account-admin/duplicate-user-error)
  </Tab>
  <Tab title="Permissions">
    Control what users can access and modify through role-based permissions.
    
    **Permission levels:**
    - Owner
    - Admin
    - Responder
    - Observer
    
    [Configure permissions →](/help-centre/account-admin/permissions/index)
  </Tab>
</Tabs>
